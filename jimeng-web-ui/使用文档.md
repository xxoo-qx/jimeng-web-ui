# 即梦 Web UI 使用文档

即梦 AI 的 Web 界面，用于文生图、图生图与视频生成。本页面说明如何部署、配置和日常使用。

---

## 一、部署方式

### 1. 仅部署 Web 前端（推荐：API 已单独部署时）

前端通过「设置」中的「API 基础地址」直连你的 jimeng-api，无需与 API 同机。

**Docker 镜像（GHCR）：**

```bash
docker run -d \
  --name jimeng-web-ui \
  -p 9901:80 \
  --restart unless-stopped \
  ghcr.io/<你的账号>/jimeng-web-ui:latest
```

浏览器访问：`http://<服务器IP>:9901`。

**本地用 Docker Compose 构建并运行：**

```bash
cd jimeng-web-ui
docker compose up -d --build
```

同样访问 `http://localhost:9901`。

**本地开发运行：**

```bash
cd jimeng-web-ui
npm ci
npm run dev
```

默认访问 `http://localhost:9901`。

---

### 2. 与 API 同机部署时的端口说明

- **jimeng-api**：默认端口 `5100`
- **jimeng-web-ui（本前端）**：容器映射为 `9901:80` 或开发时 `9901`

前端默认会使用「当前访问的主机 + 5100」作为 API 地址；若 API 不在本机或端口不同，需在设置中修改「API 基础地址」。

## 二、首次配置

### 1. 打开设置

点击页面右上角 **设置**（齿轮图标）。

### 2. API 基础地址

- 若 API 与前端在同一台机、且 API 端口为 5100：可保持默认（自动为 `http://<当前主机>:5100`）。
- 若 API 在其它机器或端口：填写完整地址，例如：
  - `http://192.168.1.100:5100`
  - `https://your-api-domain.com`

保存后，所有文生图/图生图/视频请求都会发往该地址。

### 3. Session 获取 API 地址（可选）

这是**另一个接口**，仅用于「从 API 获取」Session ID，与上面的「API 基础地址」不同：

- **API 基础地址**：jimeng-api 的地址，用于文生图、图生图、视频生成（如 `http://host:5100`）。
- **Session 获取 API 地址**：可选。填写后，在设置页点击「从 API 获取」会 **GET** 该地址，用返回的 Session 自动填入并切换为美区。

**正确使用方式：**

- **不打算用「从 API 获取」**：此处留空即可，在下方「Session ID」里手动粘贴从即梦/Dreamina 获取的 sessionid。
- **打算用「从 API 获取」**：填你使用的 **Token 管理服务**提供的、**返回账号/Session 列表**的 **GET** 接口的**完整 URL**，并在「Session 获取 API Key」中填写该服务要求的鉴权（如 Bearer Token）。  
  - **[Dreamina-Token](https://github.com/xxoo-qx/Dreamina-Token)**（xxoo-qx/Dreamina-Token）：正确地址为 **`http://<服务器>:3000/api/dreamina/getAllAccounts`**（服务默认端口 3000）。在「Session 获取 API Key」中填写 `.env` 里配置的 API 密钥（如 `sk-dreamina-admin`），请求时会自动以 `Authorization: Bearer <API_KEY>` 发送。  
  - 若报错「当前请求为 GET -> /accounts 请纠正」，说明路径填错；Dreamina-Token 应使用上述 `/api/dreamina/getAllAccounts`，不要用 `/accounts`。  
  - 其他 Token 管理服务：以该服务的 API 文档为准，填写其「获取账号列表」的 GET 完整 URL 和鉴权 Key。

**注意**：jimeng-api（iptag）没有「返回 Session 列表」的接口，**不要**把 jimeng-api 的地址（如 `http://host:5100`）填在这里。

**接口约定**：前端支持两种返回格式其一即可：
- 通用：`{ "sessionId": "xxx" }`
- 账号列表：`{ "data": [ { "sessionid": "xxx", "disabled": false } ] }`（会从可用账号中随机取一个）

不填此项则只能手动输入 Session ID。

### 4. Session ID 与区域

- **Session ID**：从即梦（国内）或 Dreamina（国际）官网登录后，在浏览器开发者工具 → 应用 → Cookie 中复制 `sessionid` 的值；或使用上方「从 API 获取」自动拉取。
- **区域**：与账号对应。
  - **国内站**：即梦，不勾选「国际站」。
  - **美国站**：Dreamina 美区，选「美国」。
  - **香港/日本/新加坡**：Dreamina 对应站点。

### 5. 保存

点击保存后刷新或切换页面即可生效。

## 三、功能使用

### 文生图

1. 侧栏进入 **文生图**。
2. 输入 **提示词**（必填）。
3. 选择 **模型**（如 jimeng-4.5、jimeng-4.1、Nanobanana 等，国际站可见 Nanobanana）。
4. 选择 **比例**（1:1、16:9 等）和 **分辨率**（1K/2K/4K，默认 2K）。
5. 可选：负面提示词、采样强度、智能比例（部分模型支持）。
6. 点击生成，等待结果（通常数十秒到数分钟，与模型和分辨率有关）。

### 图生图

1. 侧栏进入 **图生图**。
2. 上传本地图片或填写图片 URL。
3. 输入提示词并选择模型、比例、分辨率。
4. 点击生成。

### 视频生成

1. 侧栏进入 **视频生成**。
2. 选择 **视频模型**（如 jimeng-video-3.5-pro）。
3. 输入提示词；可选上传首帧/尾帧图片（图生视频或首尾帧模式）。
4. 选择比例、时长、分辨率（视模型支持）。
5. 点击生成。视频生成耗时较长，请耐心等待。

## 四、超时与错误

### 请求超时（15 分钟）

- 前端与 API 的生成请求超时时间为 **15 分钟**（与 iptag/jimeng-api 一致）。
- 若任务排队或模型较慢，可能接近 15 分钟才返回；超时会提示「请求超时，请稍后重试」。

### 504 Gateway Timeout（如 180 秒）

若出现 **504** 且报错中含 `timeout of 180000ms exceeded`（180 秒 = 3 分钟），说明是 **前端与 API 之间的某一层** 在 3 分钟就断开了，不是本前端的 15 分钟限制。

常见原因与处理：

1. **反向代理（如 Nginx）**  
   在代理到 API 的 `location` 中增大超时，例如：
   ```nginx
   proxy_connect_timeout 900s;
   proxy_send_timeout 900s;
   proxy_read_timeout 900s;
   ```
2. **API 自身或上游网关**  
   若使用自建或修改过的 jimeng-api，检查其请求上游（即梦/ Dreamina）的超时配置，将约 180 秒改为 900 秒或更长。
3. **云负载均衡/API 网关**  
   在云控制台中将「后端/上游超时」调大至至少 900 秒。

调整后重新发起生成即可。

### 其它常见错误

- **Session ID 无效或已过期**：重新从官网获取 sessionid，并在设置中选择正确区域。
- **无法连接 / 网络错误**：检查「API 基础地址」是否正确、API 是否启动、防火墙/安全组是否放行对应端口。
- **ret=1000, invalid parameter**：多为分辨率/比例与当前区域或模型不兼容（例如美区 1K 限制），可尝试换 2K 或换模型。

## 五、多区域与模型说明

- **国内站**：使用即梦账号与 Session，模型为 jimeng-* 系列。
- **国际站（美国/香港/日本/新加坡）**：使用 Dreamina 账号，Session 需带对应前缀（如美区 `us-xxx`，由设置中的区域自动处理）；国际站可使用 Nanobanana 等模型。
- 各区域可用模型、分辨率与比例以页面下拉选项为准；美区部分模型固定 2K 或固定 1024×1024，界面会做相应限制。

## 六、Docker 镜像构建（自建镜像时）

若从源码构建「仅前端」镜像（不包含 API）：

```bash
cd jimeng-web-ui
docker build -f Dockerfile.standalone -t jimeng-web-ui .
docker run -d -p 9901:80 jimeng-web-ui
```

构建时如需传入访问密码等，可使用 `Dockerfile.standalone` 中的 `ARG VITE_ACCESS_PASSWORD`，并在 `docker build` 时通过 `--build-arg` 传入。
